# <a name="list-of-prerequisite-checks-for-system-center-configuration-manager"></a>Előfeltétel-ellenőrzések listáját a System Center Configuration Managerhez

*Vonatkozik: A System Center Configuration Manager (aktuális ág)*

A következő szakaszokban az elérhető előfeltétel-ellenőrzésekről tájékozódhat.

Az előfeltétel-ellenőrző használatával kapcsolatos információkért lásd: [előfeltétel-ellenőrző](prerequisite-checker.md).  

##  <a name="BKMK_Security"></a>Biztonsági jogosultságok előfeltételeinek ellenőrzése  
A következő táblázat ellenőrzi, hogy az Előfeltételek ellenőrzője hajt végre a biztonsági jogosultságai.

|Ellenőrzés végrehajtása|MAGYARÁZAT|Severity|Hely alkalmazhatósági|
|---|---|---|---|
|**Központi adminisztrációs hely rendszergazdai jogosultságai**|Ellenőrzi a Configuration Manager telepítőjét futtató felhasználói fiók rendelkezik-e **rendszergazda** jogok a központi adminisztrációs hely számítógépén. |Hiba|Elsődleges hely|
|**A kibontott elsődleges hely rendszergazdai jogosultságai**|Ellenőrzi, hogy a telepítőt futtató felhasználói fiók rendelkezik-e **rendszergazda** jogosultságokkal a kibontásra váró, önálló elsődleges hely.|Hiba|Központi adminisztrációs hely|
|**Helyrendszer rendszergazdai jogosultságai**|Ellenőrzi, hogy a Configuration Manager telepítőjét futtató felhasználói fiók rendelkezik-e **rendszergazda** jogok a helykiszolgáló számítógépén. |Hiba|Központi adminisztrációs hely <br>Elsődleges hely <br>Másodlagos hely|
|**A kibontott elsődleges hely CAS számítógépeinek rendszergazdai jogosultságai**|Ellenőrzi, hogy rendelkezik-e a központi felügyeleti hely számítógépfiókjának **rendszergazda** jogosultságokkal a kibontásra váró, önálló elsődleges hely.|Hiba|Központi adminisztrációs hely|
|**Kapcsolódás az SQL Serverhez a központi adminisztrációs helyen**|Ellenőrzi, hogy a az elsődleges hely csatolása meglévő hierarchiához csatlakozás a Configuration Manager telepítőjét futtató felhasználói fiók rendelkezik-e a **sysadmin** szerepkör az SQL Server-példányon a központi adminisztrációs hely.|Hiba|Elsődleges hely|
|**Helykiszolgáló számítógép fióknak rendszergazdai jogai**|Ellenőrzi, hogy a helykiszolgáló számítógépfiókja rendelkezik-e **rendszergazda** jogosultságokkal az SQL server és a felügyeleti pont számítógépein.|Hiba|Elsődleges hely <br>SQL-kiszolgáló|
|**Helyrendszer és SQL Server közötti kommunikáció**| Ellenőrzi, hogy egy érvényes egyszerű szolgáltatásnév (SPN) regisztrálva van-e a helyadatbázishoz SQL Server-példányhoz, amelyen a Configuration Manager az SQL Server szolgáltatás futtatásához konfigurált fiók Active Directory tartományi szolgáltatásokban. A Kerberos-hitelesítés támogatásához érvényes egyszerű szolgáltatásnevet kell regisztrálni az Active Directory tartományi szolgáltatásokban.|Figyelmeztetés|Másodlagos hely <br>Felügyeleti pont|
|**SQL Server biztonsági üzemmód**|Ellenőrzi, hogy az SQL Server konfigurálva van-e a Windows-hitelesítési biztonsághoz.|Figyelmeztetés|SQL-kiszolgáló|
|**SQL Server rendszergazdai jogosultságok**|Ellenőrzi, hogy a Configuration Manager telepítőjét futtató felhasználói fiók rendelkezik-e a **sysadmin** szerepkört a Helyadatbázis telepítéséhez kiválasztott SQL Server-példányon. Ez az ellenőrzés akkor is sikertelen, ha a telepítő nem fér hozzá az SQL Server-példányhoz az engedélyek ellenőrzése érdekében.|Hiba|SQL-kiszolgáló|
|**SQL Server rendszergazdai jogosultságok a referenciahely számára**|Ellenőrzi, hogy a Configuration Manager telepítőjét futtató felhasználói fiók rendelkezik-e a **sysadmin** szerepkörrel a referenciahely adatbázisaként kijelölt SQL Server-példányon. SQL Server **sysadmin** szerepkörengedélyei a Helyadatbázis módosításához szükséges.|Hiba|SQL-kiszolgáló|

##  <a name="BKMK_Dependencies"></a>A Configuration Manager függőségek előfeltételeinek ellenőrzése
A következő táblázat ellenőrzi, hogy az Előfeltételek ellenőrzője hajt végre a Configuration Manager függőségek.

|Ellenőrzés végrehajtása|MAGYARÁZAT|Severity|Hely alkalmazhatósági|
|---|---|---|---|
|**A cél elsődleges helyének aktív áttelepítési leképezése**|Ellenőrzi, hogy nincs-e aktív áttelepítési leképezés elsődleges helyekre.|Hiba|Központi adminisztrációs hely|
|**Aktív replika felügyeleti pont**|Ellenőrzi az aktív felügyeletipont-replikát.|Hiba|Elsődleges hely|
|**Terjesztési pont rendszergazdai jogosultságai**|Ellenőrzi, hogy a telepítőt futtató felhasználói fiók rendelkezik-e **rendszergazda** jogosultságokkal a terjesztési pont számítógépén.|Figyelmeztetés|Terjesztési pont|
|**Felügyeleti pont rendszergazdai jogosultságai**|Ellenőrzi, hogy a helykiszolgáló számítógépfiókja rendelkezik-e **rendszergazda** jogosultságokkal a felügyeleti pont és a terjesztési pont számítógépén.|Figyelmeztetés|Felügyeleti pont|
|**Rendszergazdai megosztás (helyrendszer)**|Ellenőrzi, hogy a helyrendszer számítógépén jelen-e a szükséges rendszergazdai megosztások.|Figyelmeztetés|Felügyeleti pont|
|**Alkalmazáskompatibilitás**|Ellenőrzi, hogy aktuális alkalmazások megfelelnek az alkalmazási sémának.|Figyelmeztetés|Központi adminisztrációs hely <br>Elsődleges hely|
|**BITS-támogatás**|Ellenőrzi, hogy a háttérben futó intelligens átviteli szolgáltatás (BITS) telepítve van a felügyeleti pont helyrendszerének számítógépére. Ha ez az ellenőrzés sikertelen, a BITS nincs telepítve, az Internet Information Services (IIS) 6.0 Windows Management Instrumentation (WMI) kompatibilitási összetevő az IIS 7.0 nincs telepítve a számítógépen, vagy a távoli IIS állomáson, vagy telepítője nem tudta ellenőrizni a távoli IIS-beállításokat, mert az IIS közös összetevőit nem telepítették a helykiszolgáló számítógépén.|Hiba|Felügyeleti pont|
|**BITS telepítve**|Ellenőrzi, hogy a BITS telepítve van-e az IIS-ben.|Figyelmeztetés|Felügyeleti pont|
|**Nem betűérzékeny rendezés az SQL Server**|Ellenőrzi, hogy az SQL Server-telepítés például az SQL_Latin1_General_CP1_CI_AS a nagybetűk között rendezést használ-e.|Hiba|SQL-kiszolgáló|
|**Meglévő önálló elsődleges helyek verziójának és helykódjának ellenőrzése**|Ellenőrzi, hogy a kibontani kívánt elsődleges hely önálló elsődleges hely, és arról, hogy rendelkezik-e a Configuration Manager, de eltérő helykóddal ugyanazon verzióját mint a telepítendő központi adminisztrációs hely.|Hiba|Központi adminisztrációs hely <br>Elsődleges hely|
|**Inkompatibilis gyűjtemények hivatkozásainak keresése**|A frissítés során ez ellenőrzi, hogy a gyűjtemények csak más azonos típusú gyűjteményekre hivatkoznak.|Hiba|Központi adminisztrációs hely|  
|**Ügyfélverzió a felügyeleti pont számítógépén**|Ellenőrzi, hogy telepíti a felügyeleti pont egy számítógépen, amelyen nincs telepítve a Configuration Manager-ügyfél egy másik verziója.|Hiba|Felügyeleti pont|
|**Az SQL Server memóriahasználatának konfigurálása**|Ellenőrzi, hogy az SQL Server korlátlan memóriahasználatot konfigurálták. Az SQL Server memóriájához maximális korlátot kell megadnia.|Figyelmeztetés|SQL-kiszolgáló|
|**Dedikált SQL Server-példány**|Ellenőrzi, hogy dedikált SQL server-példányt konfiguráltak-e a Configuration Manager-Helyadatbázis üzemeltetésére. Ha egy másik hely használja az adott példányt, akkor az új hely számára egy másik példányt kell választania. Alternatív megoldásként távolítsa el a másik hely vagy az adatbázisát áthelyezheti egy másik SQL server-példányt.|Hiba|Központi adminisztrációs hely <br>Elsődleges hely <br>Másodlagos hely|
|**Meglévő Configuration Manager kiszolgáló-összetevők a kiszolgálón**|Ellenőrzi, hogy a helykiszolgáló vagy helyrendszer-szerepkör nem már telepítve van a hely telepítéséhez kijelölt számítógépre.|Hiba|Központi adminisztrációs hely <br>Elsődleges hely <br>Másodlagos hely|
|**Az SQL Server tűzfalkivételének ellenőrzése**|Ellenőrzi, hogy a Windows tűzfala le van tiltva, vagy hogy a megfelelő Windows tűzfalkivétel van-e az SQL Server. Alapértelmezés szerint az SQL Server a 1433-as TCP-porton figyel, és az SQL Server Service Broker (SSB) 4022-es portot használja.|Hiba|Központi adminisztrációs hely <br>Elsődleges hely <br>Másodlagos hely <br>Felügyeleti pont|
|**Tűzfal-kivétel az SQL Server (önálló elsődleges hely)**|Ellenőrzi, hogy a Windows tűzfala le van tiltva, vagy hogy a megfelelő Windows tűzfalkivétel van-e az SQL Server. Engedélyezni kell az Sqlservr.exe vagy a szükséges TCP-portok távoli elérését. Alapértelmezés szerint az SQL Server a 1433-as TCP-portot figyeli, és az SSB 4022-es portot használja.|Figyelmeztetés|Elsődleges hely (csak önálló)|
|**A felügyeleti pont az SQL Server tűzfalkivételének ellenőrzése**|Ellenőrzi, hogy a Windows tűzfala le van tiltva, vagy hogy a megfelelő Windows tűzfalkivétel van-e az SQL Server.|Figyelmeztetés|Felügyeleti pont|
|**IIS HTTPS-konfiguráció**|Ellenőrzi az IIS-webhelykötéseket a HTTPS kommunikációs protokollhoz. Ha HTTPS PROTOKOLLT igénylő helyszerepköröket telepít, konfigurálnia kell az IIS-helykötéseket érvényes nyilvános kulcsokra épülő infrastruktúra (PKI) tanúsítványt a megadott kiszolgálón.|Figyelmeztetés|Felügyeleti pont <br>Terjesztési pont|
|**IIS-szolgáltatás fut**|Ellenőrzi, hogy az IIS telepítve és fut a számítógépen, hogy a felügyeleti pont és terjesztési pont telepítéséhez.|Hiba|Felügyeleti pont <br> Terjesztési pont|
|**Kibontott elsődleges hely illesztésének megfeleltetése**|Ellenőrzi, hogy a bővíteni kívánt önálló elsődleges hely adatbázis rendelkezik-e a központi adminisztrációs helyen a Helyadatbázis rendezésével.|Hiba|Központi adminisztrációs hely|
|**Microsoft Távoli különbözeti tömörítés (RDC) függvénytára regisztrálva**|Ellenőrzi, hogy az RDC függvénytára regisztrálva van-e a Configuration Manager helykiszolgálón.|Hiba|Központi adminisztrációs hely <br>Elsődleges hely <br>Másodlagos hely|
|**Microsoft Windows Installer**|Ellenőrzi a Windows Installer verziószámát. Ha ez az ellenőrzés sikertelen, a telepítő nem tudta ellenőrizni a verziószámot, vagy jelenleg telepített változata nem felel meg a Windows Installer 4.5-ös verziójának minimális követelményeit.|Hiba|Központi adminisztrációs hely <br>Elsődleges hely <br>Másodlagos hely|
|**Microsoft XML Core Services 6.0 (MSXML60)**|Ellenőrzi, hogy az MSXML 6.0 vagy újabb verziója telepítve van a számítógépen.|Figyelmeztetés|Központi adminisztrációs hely <br>Elsődleges hely <br>Másodlagos hely <br>A Configuration Manager konzol <br>Felügyeleti pont <br>Terjesztési pont|
|**.NET-keretrendszer minimális verziója a Configuration Manager konzol**|Ellenőrzi, hogy a Microsoft .NET-keretrendszer 4.0-s verzióját a Configuration Manager konzol számítógépen van-e telepítve. A .NET-keretrendszer 4.0 letölthető [Microsoft Download Center](http://go.microsoft.com/fwlink/p/?LinkId=189149).|Hiba|Configuration Manager-konzol|
|**.NET-keretrendszer minimális verziója a Configuration Manager-helykiszolgáló**|Ellenőrzi, hogy a .NET-keretrendszer 3.5 telepítve van a Configuration Manager helykiszolgálón. A Windows Server 2008, töltse le a Microsoft .NET-keretrendszer 3.5-ös verzióját a [Microsoft Download Center](http://go.microsoft.com/fwlink/p/?LinkId=185604). A Windows Server 2008 R2 a .NET-keretrendszer 3.5 engedélyezheti szolgáltatásként a Kiszolgálókezelőből.|Hiba|Központi adminisztrációs hely <br>Elsődleges hely <br>Másodlagos hely|
|**.NET-keretrendszer minimális verziója az SQL Server Express kiadásának telepítéséhez a Configuration Manager másodlagos hely**|Ellenőrzi, hogy a .NET Framework 4.0 telepítve van-e a Configuration Manager másodlagos helyéhez tartozó SQL Server Express telepítése számítógépekre.|Hiba|Másodlagos hely|
|**Szülőadatbázis-/ gyermekadatbázis-rendezés**|Ellenőrzi, hogy a Helyadatbázis rendezése azonos-e a szülőhely adatbázisának rendezésével. A hierarchiában található összes helyen ugyanazt az adatbázis-rendezést kell alkalmazni.|Hiba|Elsődleges hely <br>Másodlagos hely|

|**Elsődleges FQDN**|Egy teljesen minősített tartománynevét (FQDN) használatával ellenőrzi, hogy a számítógép NetBIOS-neve egyezik-e a helyi állomásnevével (az FQDN első címkéje) a számítógép.|Hiba|Központi adminisztrációs hely <br>Elsődleges hely <br>Másodlagos hely <br>SQL-kiszolgáló|
